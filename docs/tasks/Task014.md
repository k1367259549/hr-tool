# API Endpoints

## Generate Plan

```http
POST /api/planner/generate
```

Request:

```json
{
  "date": "2026-01-02"
}
```

Behavior:

- Find previous day RecruitLog
- Find previous day DailyReview if available
- Build prompt using `/prompts/planner.md`
- Call AI service
- Parse JSON output
- Validate schema
- Save DailyPlan
- Return saved plan

## Get Plan by Date

```http
GET /api/planner/date/:date
```

---

# Planner Output Schema

The canonical V1 schema follows `/docs/07_AI_DESIGN.md` and the current Prisma `DailyPlan` model:

```json
{
  "date": "2026-01-02",
  "schedule": [
    {
      "time": "morning",
      "content": "Screen resumes for priority roles",
      "priority": "HIGH"
    }
  ],
  "priorityTasks": ["Screen resumes for priority roles"],
  "goals": ["Improve interview conversion"],
  "risks": ["Insufficient qualified resume volume"],
  "expectedOutcomes": ["Shortlist qualified candidates"],
  "priority": "HIGH"
}
```

---

# Validation Rules

- date is required
- schedule must not be empty
- time must be one of:
  - morning
  - afternoon
  - evening
- priority must be one of:
  - LOW
  - MEDIUM
  - HIGH
