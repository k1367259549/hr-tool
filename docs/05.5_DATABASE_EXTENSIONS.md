# Database Extensions for V1

Version: V1.0
System: HR Daily AI

---

# 1. Purpose

This document defines optional but recommended database extensions for V1.

These extensions are designed to improve future scalability without adding major complexity.

---

# 2. Recommended Additions

## 2.1 Extend RecruitLog

Add optional fields:

```prisma
source        String?
channel       String?
roleType      String?
priority      String?
```

Meaning:

- source: candidate source, e.g. Boss, LinkedIn, Referral
- channel: recruiting channel
- roleType: job type, e.g. technical, product, HR
- priority: LOW / MEDIUM / HIGH

---

# 3. Add AIRequestLog

Purpose:

Track AI API usage, response time, errors, and future cost analysis.

Prisma model:

```prisma
model AIRequestLog {
  id           String   @id @default(uuid())
  feature      String
  model        String
  promptTokens Int?
  outputTokens Int?
  totalTokens  Int?
  latencyMs    Int?
  success      Boolean
  errorMessage String?
  createdAt    DateTime @default(now())
}
```

---

# 4. AI Output Storage Rule

AI output should be stored in two formats:

- structured JSON -> database analysis
- markdown text -> user display/export

For V1, JSON is mandatory. Markdown is optional but recommended.

These extensions may be implemented during Task003 if not yet completed.

---

# 6. Out of Scope

Do not implement:

- vector database
- embedding storage
- RAG
- multi-user memory
- long-term agent memory system

---

# 7. Summary

These extensions improve:

- future analytics
- AI cost tracking
- recruiting source analysis
- better AI-generated reports
